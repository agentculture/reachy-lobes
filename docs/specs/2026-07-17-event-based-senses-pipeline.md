# event-based senses pipeline

> Reachy's live presence becomes a fully event-based senses pipeline: speech is captured whole and heard correctly (continuous recording with a pre-roll ring buffer that reaches 1-2 seconds back at speech onset, ported from reachy_nova's proven design), and every sense — spoken words, audio direction, touch, camera images, and basic face recognition — lands as an event the agent reasons over; every pipeline stage emits a structured log line so capture can be verified end-to-end; the agent responds seamlessly by speaking, playing its harmonic voice, moving its body, or all at once; and code tasks flow to qwen3 to forge new reaction seams for the agent to use or improve
> instruction: build as three PRs by concern (user decision): PR A hearing pre-roll + per-stage [SENSE] logging + handler install; PR B camera-path repair + vision/face/scene events behind the new [vision] extra; PR C the qwen3 forge (validator-gated auto-activate); each PR live-verified on the robot and version-bumped; land after the in-flight 66/67/68 PRs

## Audience

- the person in the room with the robot (their sentences are heard whole from the first word, their face is known, their touch is felt), the operator debugging the pipeline (every stage visible in the journal), the agent model itself (a fuller sensorium of events in one buffer), and the qwen3 coder rig acting as the seam-smith
  - instruction: verify per audience in the live acceptance session: person (whole-sentence hearing, name greeting, touch response), operator (journal trail), model (new cue kinds in its buffer), rig (real forge request served)

## Before → After

- Before: utterances reach STT clipped of their leading words (capture starts on a lagging 5Hz flag, no pre-roll), vision never reaches cognition (feed_vision has zero production callers), faces are anonymous (no face code exists), per-stage INFO traces are silently dropped (no logging handler anywhere) so the live pipeline is undebuggable, and the seam library (stash) is wired to nothing
  - instruction: cite the evidence lines in the spec (listen_transcribe.py:296 gate, feed_vision zero callers, face grep empty, no logging handler, stash unimported) so the before-state stays checkable
- After: the robot hears sentences whole (pre-roll capture from before the speech flag flips), knows who walked in (face event with a name), sees motion and light change (vision cues), feels touch (inherited from 66/67/68) — every perception an event in the one cognition buffer; the agent answers by speaking, playing harmonics, moving, or all three in one turn; an operator can tail journalctl and watch one perception travel capture -> event -> cue -> turn -> action through structured [SENSE] lines; and the agent can commission qwen3 to forge a new reaction seam that passes validation and becomes usable
  - instruction: demonstrate the full after-state in one on-robot session and keep a transcript; each capability also has a no-robot test proving its seam

## Why it matters

- an agent is only as rich as its sensorium and only as trustworthy as its observability — 66/67/68 gave it touch; this gives it whole sentences, sight, and identity, makes capture verifiable in the journal instead of assumed, and opens the path for the agent to grow its own reactions

## Assumptions

- no VAD library is needed or added: reachy_nova ships none (no webrtcvad/silero in its pyproject or uv.lock) — onset is ASR-driven or the XMOS hardware speech flag, and reachy-mini-cli already reads that same flag as sense.speech_detected; the pre-roll buffer is what makes the 5Hz flag lag harmless, not a better detector
- the seamless multi-modal response already exists and needs verification, not construction: ToolRegistry registers speak (TTS), harmonics (melodic voice), and apply_pose (catalog enum) as independent tools, dispatch never raises, and AgentTurnEngine runs up to 6 tool rounds per turn — so speak-or-harmonics-or-move-or-all is the agent picking several tools in one turn, already supported

## Decisions

- face recognition runs on-box: OpenCV YuNet+SFace in a NEW [vision] extra (opencv-python-headless), models auto-downloaded, missing extra raises the established exit-2 CliError (user decision resolving q1)
- scene description ships this pass in BOTH modes — an on-demand describe tool and periodic description events — backed by Gemma4 via the lobes role endpoints: senses (Gemma4-12B, loaded) as default, muse (Gemma4-31B) as the heavier option (user decision resolving q2)
- the qwen3 forge produces generated code (SKILL.md + executor), gated by the ported AST-only fail-closed validator, and AUTO-activates on pass — no human gate, matching the deliberate nova decision (user decision resolving q3)
- new sense events (face, vision, scene) bypass decide_engagement like pat cues; flood control is per-sense cooldowns (face re-announce ~30s, rate-limited direction, one vision cue per motion episode) (user decision resolving q4)
